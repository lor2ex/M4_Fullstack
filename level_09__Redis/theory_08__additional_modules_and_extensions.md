На момент конца 2025 года архитектура Redis значительно эволюционировала:  
начиная с версии **Redis 8.0** (выпущена в 2025 году),  
прежний отдельный пакет **Redis Stack** интегрирован напрямую в **Redis Open Source**.  
Отдельные модули (как RediSearch, RedisJSON, RedisTimeSeries, RedisBloom) больше не требуют отдельной установки  
— их функциональность встроена в основной сервер Redis и доступна по умолчанию.

### Блок 7: Дополнительные модули и расширения

#### Основные встроенные расширения в Redis 8+
   - **JSON**: 
     - Полноценная поддержка хранения, обновления и запросов JSON-документов как нативного типа данных.
     - Ключевые возможности: 
       - пути в стиле JSONPath для доступа к элементам (например, `.path.to.field`), 
       - атомарные обновления, 
       - индексация полей.
     - Применение: 
       - хранение сложных объектов (документная модель), 
       - семантический кэш, feature stores в ML. 
       - В Python: 
         - команды вроде `JSON.SET`, `JSON.GET`
         - доступны через стандартный клиент redis-py без дополнительных библиотек.
   
   - **Поиск и запросы (Search and Query Engine)**:
     - Полнотекстовый поиск, вторичные индексы, агрегации, фильтры.
     - Поддержка векторного поиска (KNN для similarity), гибридных запросов (текст + векторы).
     - Применение: 
       - поиск по данным, рекомендации, RAG-системы в AI. 
       - Команды: `FT.CREATE` (индекс), `FT.SEARCH`, `FT.AGGREGATE`.
   
   - **Временные ряды (Time Series)**:
     - Специализированный тип для хранения и анализа timestamped данных.
     - Высокая производительность ingestion, downsampling, агрегации (avg, min/max, count), retention policies.
     - Применение: 
       - мониторинг метрик, IoT, финансовые данные. 
       - Команды: `TS.CREATE`, `TS.ADD`, `TS.RANGE`, `TS.MRANGE`.
    
   - **Векторные множества (Vector Sets)**: 
     - Поддержка хранения и поиска векторов (embeddings), включая KNN-поиск.

#### Устаревшие или удаленные модули
   - **RedisGraph**: 
     - Полностью прекращена поддержка с января 2025 года. 
     - Графовые запросы (Cypher) больше не рекомендуются; 
     - для графов предлагается использовать внешние решения или комбинировать с другими типами данных.
