### 1 Краткое определение

* **SQL (реляционные базы данных)**: 
  * базы данных с фиксированной схемой, хранящие данные в связанных таблицах.
* **NoSQL (нереляционные базы данных)**: 
  * базы данных без фиксированной схемы, 
  * данные могут храниться в документах, графах, ключ-значение или колонках.

---

### 2 Плюсы и минусы

**SQL**  

   * **Плюсы**
     * Надёжность и консистентность (ACID)
     * Строгая структура данных 
     * Валидация данных уровне самой структуры
     * Компактность (благодаря нормализации данных)
     * Мощные SQL-запросы

   * **Минусы**
     * Сложность масштабирования горизонтально 
     * Высокая стоимость обслуживания больших БД (из-за вертикального масштабирования)
     * Изменения структуры требуют опыта и усилий
       
**NoSQL**  
   * **Плюсы**
   * Простая горизонтальная масштабируемость 
   * Низкая стоимость при увеличении объёма БД
   * Гибкая схема 
     * Структура данных меняется "на лету"
   * Высокая производительность для больших данных 
     * За счёт отсутствия JOINов

   * **Минусы**
     * Меньше гарантий консистентности
       * Данные в репликах могут "не успевать" за изменениями на Primary Node
     * Ограниченные возможности сложных запросов
       * Отсюда неоптимальное (избыточное) хранение данных
---

### 3 Области применения

**SQL**

* **Финансы** 
  * банки, 
  * платёжные системы, 
  * бухгалтерия (критичная консистентность)
* **ERP и CRM-системы** — управление ресурсами и клиентами с жёсткой структурой данных
* **Классические веб-приложения**
  * сайты и сервисы с предсказуемой схемой данных 
  * и транзакциями
* **Отчётность и аналитика с высокими требованиями к точности**

---

**NoSQL**

* **Социальные сети** 
  * хранение больших объёмов неструктурированных пользовательских данных, 
  * быстрый доступ
* **IoT и сенсорные данные**
  * потоковые данные с высокой частотой 
  * и разной (нестрогой) структурой
* **Big Data и real-time analytics**
  * обработка огромных массивов данных с высокой скоростью
* **Контентные платформы и высоконагруженные сервисы**
  * гибкая структура
  * масштабирование «по требованию»

---

### 4 Итого

| Характеристика  | SQL                                            | NoSQL                                                                     |
| --------------- | ---------------------------------------------- | ------------------------------------------------------------------------- |
| **Определение** | Реляционная БД с таблицами и строгой схемой    | Нереляционная БД без фиксированной схемы                                  |
| **Плюсы**       | ACID, строгая структура, мощные запросы        | Горизонтальная масштабируемость, гибкая схема, высокая производительность |
| **Минусы**      | Сложное масштабирование, жёсткая схема         | Ограниченная консистентность, сложные запросы                             |
| **Применение**  | Финансы, ERP, CRM, классические веб-приложения | Соцсети, IoT, big data, real-time analytics, высоконагруженные сервисы    |
